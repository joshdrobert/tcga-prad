# Lost in Translation: How Endpoint Discordance Undermines Genomic Prediction of Prostate Cancer Recurrence

This repository contains the standalone code and manuscript materials for the publication:
**"Endpoint Harmonization and Internal Baseline Modeling for Recurrence/Progression in Open TCGA-PRAD Clinical Data"**

## Repository Layout
- `src/`: Core analysis scripts (data construction, training, validation).
- `manuscript/`: Self-contained LaTeX manuscript source code.
- `tables/`: Summary tables exported from pipeline runs.
- `figures/`: High-resolution figures generated by inline TikZ/PGFPlots or python code.
- `data_processed/`: Intermediate processed data cohorts.
- `data_raw/`: Raw data manifests and instructions.
- `_shared/`: Shared helper functions and environment dependencies.

## Installation & Setup
To run the code, set up a clean Python 3.10+ virtual environment and install the required dependencies:

```bash
# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate  # On Windows: .\.venv\Scripts\Activate.ps1

# Install requirements
pip install -r _shared/00_environment/requirements.txt
```

## Data Access
Requires open clinical XML supplements from the GDC portal. Manifests and precheck lists are included under data_raw/.

## Running the Reproduction Pipeline
Run the scripts from the repository root:

```bash
python src/build_available_data.py
python src/build_recurrence_endpoint.py
python src/cbioportal_endpoint_crosscheck.py
python src/train_recurrence_model.py
python src/run_rigorous_validation.py
```

All figures and tables will be generated directly in the `figures/` and `tables/` directories, ready to compile with the LaTeX manuscript in `manuscript/manuscript.tex`.
